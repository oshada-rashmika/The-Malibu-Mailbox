# Security Policy

## Supported Versions

Currently, the primary maintained version of The Malibu Mailbox is the `main` branch and the latest deployment. We only backport security updates to the current active release.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.x | :x:                |

## Reporting a Vulnerability

We take the security of The Malibu Mailbox seriously. If you believe you have found a security vulnerability, **please do not report it through public GitHub issues**, as this can expose the vulnerability to malicious actors before a patch is ready.

### How to Report

Please report security vulnerabilities privately using GitHub's built-in reporting tool:

1. Navigate to the **Security** tab of this repository.
2. Select **Advisories** from the left sidebar.
3. Click the **Report a vulnerability** button.
4. Provide a detailed description of the vulnerability. Please include:
   - Steps to reproduce the issue.
   - The potential impact of the vulnerability (e.g., how an attacker could exploit it).
   - Any suggested mitigations or fixes.

*(Alternatively, you can email the maintainer directly at damien.desilvar@gmail.com with the subject line `[SECURITY VULNERABILITY] - The Malibu Mailbox`.)*

### What to Expect

- **Acknowledgement:** We will acknowledge receipt of your vulnerability report within 48 hours.
- **Updates:** We will send you regular status updates as we investigate the issue and develop a fix.
- **Resolution:** Once the vulnerability is patched, we will coordinate a timeline with you for public disclosure. We are happy to give you credit for the discovery in the release notes or security advisory if you wish.

Thank you for helping us keep The Malibu Mailbox secure!
